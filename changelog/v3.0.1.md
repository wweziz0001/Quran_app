# [3.0.1] - 2026-03-02

## 🔧 الإصلاحات

### تحديث سكربت التشغيل التلقائي (setup_and_run.sh)

**المشكلة:**
سكربت `setup_and_run.sh` لم يكن يشغل جميع الخدمات المصغرة (8 خدمات)، وكان يعاني من:
- مشاكل في التنقل بين المجلدات باستخدام `cd` النسبي
- عدم وجود سجلات منفصلة لكل خدمة
- عدم وجود فحص صحة الخدمات بعد التشغيل
- عدم وجود طريقة لإيقاف جميع الخدمات

**السبب:**
السكربت القديم صُمم قبل إضافة الخدمات المصغرة، ولم يُحدّث ليشملها.

**الحل:**
إعادة كتابة السكربت بالكامل مع:
1. استخدام مسارات مطلقة بدلاً من النسبية
2. دالة `start_service()` لتشغيل كل خدمة بشكل موحد
3. مجلد سجلات منفصل `/logs`
4. فحص صحة جميع الخدمات بعد التشغيل
5. إنشاء سكربت `stop_all.sh` لإيقاف الخدمات

**الملفات المتأثرة:**
- `setup_and_run.sh` - إعادة كتابة كاملة 🟢
- `stop_all.sh` - ملف جديد 🟢

**التفاصيل:**

### setup_and_run.sh:
```bash
# تشغيل جميع الخدمات الـ 8 مع التطبيق الرئيسي
nohup bun run dev > output.log 2>&1 &

cd services/quran-service
bun install
nohup bun run dev > output.log 2>&1 &

cd ../../services/audio-service
bun install
nohup bun run dev > output.log 2>&1 &

# ... باقي الخدمات
```

**النتيجة:**
- ✅ تشغيل جميع الخدمات الـ 8 + التطبيق الرئيسي
- ✅ سكربت إيقاف جميع الخدمات `stop_all.sh`

**الهدف:**
توفير طريقة سهلة لإيقاف جميع الخدمات دفعة واحدة.

**الملف الجديد:**

**طريقة الاستخدام:**

**الميزات:**

- عرض حالة كل خدمة قبل الإيقاف
