### إعادة بناء لوحة الإدارة - صفحات منفصلة بدلاً من صفحة واحدة

**المشكلة:**
لوحة الإدارة `/admin` كانت تعاني من عدة مشاكل:
1. بطء شديد في التحميل - جميع المكونات (14 مكون) تُحمّل في صفحة واحدة
2. عند الضغط على Refresh، يعود المستخدم للصفحة الرئيسية (Dashboard)
3. لا يوجد URL خاص بكل تبويب
4. لا يوجد Lazy Loading أو Code Splitting
5. الـ Bundle كبير جداً بسبب تحميل كل شيء معاً

**السبب:**
- كانت جميع التبويبات عبارة عن components تُحمّل داخل صفحة واحدة
- استخدام `useState` للتنقل بدلاً من Next.js Router
- جميع الاستيرادات تتم في بداية الملف

**الحل:**
تحويل كل تبويب إلى صفحة مستقلة مع layout مشترك:


## 🔧 الإصلاحات

### إصلاح خطأ Module not found: sql.js

**المشكلة:**
بعد تقسيم لوحة الإدارة، ظهر خطأ `Module not found: Can't resolve 'sql.js'` في API endpoints الخاصة بـ image-mushaf و ttf-mushaf.

**السبب:**
`sql.js` مكتبة WASM مصممة للمتصفح، وهي غير متوافقة مع Turbopack في Next.js 16.

**الحل:**
استبدال `sql.js` بـ `better-sqlite3` وهي مكتبة native Node.js تعمل بشكل أفضل على الخادم.

**الملفات المتأثرة:**
- `src/app/api/admin/image-mushaf/route.ts` - تغيير الاستيراد
- `src/app/api/admin/ttf-mushaf/route.ts` - تغيير الاستيراد
- `package.json` - إضافة better-sqlite3

**التفاصيل:**
```diff
- import initSqlJs from 'sql.js';
+ import Database from 'better-sqlite3';

- const SQL = await initSqlJs();
- const fileBuffer = await readFile(dbPath);
- const sqlDb = new SQL.Database(new Uint8Array(fileBuffer));
+ const sqlDb = new Database(dbPath, { readonly: true, fileMustExist: true });

- const surahs = sqlDb.exec('SELECT * FROM surahs');
+ const surahs = sqlDb.prepare('SELECT * FROM surahs').all() as any[];
```

**النتيجة:**
- ✅ تم إصلاح خطأ البناء
- ✅ أداء أفضل (better-sqlite3 أسرع)
- ✅ توافق كامل مع Turbopack

**مقارنة الأداء:**

| المقياس | قبل | بعد |
|---------|-----|-----|
| تحميل /admin | ثقيل (كل المكونات) | خفيف (Dashboard فقط) |
| Refresh | يعود للـ Dashboard | يبقى في نفس الصفحة |
| URL | ثابت /admin | مختلف لكل تبويب |
| Lazy Loading | ❌ | ✅ |
| Code Splitting | ❌ | ✅ |
