### إعادة بناء لوحة الإدارة - صفحات منفصلة بدلاً من صفحة واحدة

**المشكلة:**
لوحة الإدارة `/admin` كانت تعاني من عدة مشاكل:
1. بطء شديد في التحميل - جميع المكونات (14 مكون) تُحمّل في صفحة واحدة

- كانت جميع التبويبات عبارة عن components تُحمّل داخل صفحة واحدة

## 🔧 الإصلاحات

### إصلاح خطأ Module not found: sql.js

**المشكلة:**
بعد تقسيم لوحة الإدارة، ظهر خطأ `Module not found: Can't resolve 'sql.js'` في API endpoints الخاصة بـ image-mushaf و ttf-mushaf.

**السبب:**
`sql.js` مكتبة WASM مصممة للمتصفح، وهي غير متوافقة مع Turbopack في Next.js 16.

**الحل:**
استبدال `sql.js` بـ `better-sqlite3` وهي مكتبة native Node.js تعمل بشكل أفضل على الخادم.

**الملفات المتأثرة:**
- `src/app/api/admin/image-mushaf/route.ts` - تغيير الاستيراد
- `src/app/api/admin/ttf-mushaf/route.ts` - تغيير الاستيراد
- `package.json` - إضافة better-sqlite3

**التفاصيل:**
```diff
- import initSqlJs from 'sql.js';
+ import Database from 'better-sqlite3';

- const SQL = await initSqlJs();
- const fileBuffer = await readFile(dbPath);
- const sqlDb = new SQL.Database(new Uint8Array(fileBuffer));
+ const sqlDb = new Database(dbPath, { readonly: true, fileMustExist: true });

- const surahs = sqlDb.exec('SELECT * FROM surahs');
+ const surahs = sqlDb.prepare('SELECT * FROM surahs').all() as any[];
```

**النتيجة:**
- ✅ تم إصلاح خطأ البناء
- ✅ أداء أفضل (better-sqlite3 أسرع)
- ✅ توافق كامل مع Turbopack

## 🔧 الإصلاحات (تابع)

### إصلاح تبويب Schema في Database Management

**المشكلة:**
تبويب Schema في Database Management لا يظهر محتويات الجداول ولا العلاقات.

**السبب:**
API `/api/admin/db/tables` كان يعيد `columns: []` فارغة دائماً بسبب تبسيط سابق لحل مشكلة البطء. هذا أدى إلى:
- SchemaVisualizer لا يستطيع عرض الحقول
- لا تظهر العلاقات بين الجداول
- الـ nodes فارغة بدون محتوى

**الحل:**
تحديث API ليعيد الـ columns من Prisma Schema بدلاً من قاعدة البيانات مباشرة (أسرع بكثير):
- قراءة `prisma/schema.prisma` وتحليله
- استخراج الحقول مع الـ primary keys
- استخراج الـ foreign keys من العلاقات
- استخراج الـ nullable من `?` في الحقول

**الملفات المتأثرة:**
- `src/app/api/admin/db/tables/route.ts` - إعادة كتابة كاملة

**التفاصيل:**
```typescript
// قبل: columns فارغة
{
  name: "Ayah",
  columns: [],  // فارغة!
}

// بعد: columns كاملة مع foreign keys
{
  name: "Ayah",
  columns: [
    { name: "id", type: "Int", primaryKey: true, nullable: false },
    { name: "surahId", type: "Int", nullable: false, 
      foreignKey: { table: "Surah", column: "id" } },
    // ...
  ]
}
```

**النتيجة:**
- ✅ تبويب Schema يعرض جميع الجداول مع الحقول
- ✅ العلاقات تظهر بشكل صحيح (23 علاقة)
- ✅ أداء سريع (48ms بدلاً من ثواني)
- ✅ Primary Keys و Foreign Keys محددة

**مقارنة الأداء:**

| المقياس | قبل | بعد |
|---------|-----|-----|
| تحميل /admin | ثقيل (كل المكونات) | خفيف (Dashboard فقط) |
| Refresh | يعود للـ Dashboard | يبقى في نفس الصفحة |
| URL | ثابت /admin | مختلف لكل تبويب |
| Lazy Loading | ❌ | ✅ |
| Code Splitting | ❌ | ✅ |
