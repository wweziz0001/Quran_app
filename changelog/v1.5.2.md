## 🔧 الإصلاحات

### إصلاح Schema Visualizer لعرض الجداول الجديدة

**المشكلة:**
الجداول الجديدة (search_indices, analytics, notifications, etc.) لم تكن تظهر في تبويب Schema مع columns وعلاقاتها.

**السبب:**
- API `/api/admin/db/tables` كان يبحث عن columns باستخدام اسم الـ Model (مثل `SearchIndex`) بينما في قاعدة البيانات الجداول تستخدم `@@map` بأسماء مختلفة (مثل `search_indices`).
- API `/api/admin/db/schema` لم يقرأ الـ `@@map` بشكل صحيح.

**الحل:**

#### 1. تحديث `/api/admin/db/tables/route.ts`
- قراءة الـ `@@map` من كل model
- تخزين الـ columns تحت اسم الجدول الفعلي (من `@@map`)

```typescript
// Get actual table name from @@map or use model name
let tableName = modelName;
const mapMatch = modelBody.match(/@@map\(["']([^"']+)["']\)/);
if (mapMatch) {
  tableName = mapMatch[1];
}

// Store columns under the ACTUAL table name
tableToColumns.set(tableName, columns);
```

#### 2. تحديث `/api/admin/db/schema/route.ts`
- إضافة mapping بين أسماء الـ Models وأسماء الجداول
- تصدير `fromTable` و `toTable` كأسماء الجداول الفعلية

**النتيجة:**
- ✅ 42 جدول مع جميع الـ columns
- ✅ 34 علاقة بين الجداول (كانت 23)
- ✅ الجداول الجديدة تظهر مع columns وعلاقاتها:
  - `search_indices` → Ayah
  - `notifications` → User
  - `analytics` → User, Ayah
  - `user_sessions` → User
  - `audio_files` → Recitation, Ayah
  - `collections` → User
  - `collection_items` → collections, Ayah
  - `word_analyses` → Ayah
