## ✨ ميزة جديدة

### إضافة سكربت إعداد وتشغيل تلقائي (setup_and_run.sh)

**الهدف:**
تسهيل عملية إعداد وتشغيل المشروع للمطورين الجدد، بحيث يمكنهم تشغيل المشروع بأمر واحد فقط.

**الميزات:**
1. تثبيت Bun تلقائياً إذا لم يكن مثبت
2. تحميل متغيرات البيئة وجعلها دائمة في ~/.bashrc
3. إنشاء ملف .env تلقائياً مع DATABASE_URL
4. تثبيت جميع الاعتماديات المطلوبة
5. توليد Prisma Client
6. تطبيق قاعدة البيانات
7. تشغيل خادم التطوير تلقائياً

**الملف الجديد:**
- `setup_and_run.sh` - سكربت شل للإعداد والتشغيل التلقائي (قابل للتنفيذ)

**التفاصيل:**
```bash
#!/bin/bash

set -e  # إيقاف السكربت عند أي خطأ

# 1. تثبيت Bun إذا لم يكن مثبت
if ! command -v bun &> /dev/null; then
    curl -fsSL https://bun.sh/install | bash
fi

# 2. تحميل متغيرات البيئة
export BUN_INSTALL="$HOME/.bun"
export PATH="$BUN_INSTALL/bin:$PATH"

# 3. جعلها دائمة في ~/.bashrc
grep -q 'BUN_INSTALL' ~/.bashrc || echo 'export BUN_INSTALL="$HOME/.bun"' >> ~/.bashrc
grep -q 'BUN_INSTALL/bin' ~/.bashrc || echo 'export PATH="$BUN_INSTALL/bin:$PATH"' >> ~/.bashrc

# 4. التحقق من ملف .env وإنشاؤه إذا لزم الأمر
# 5. تثبيت الاعتماديات
bun install

# 6. توليد Prisma Client
bunx prisma generate

# 7. تطبيق قاعدة البيانات
bun run db:push

# 8. تشغيل خادم التطوير
bun run dev
```

**طريقة الاستخدام:**
```bash
# تشغيل مباشر
bash setup_and_run.sh

# أو جعله executable ثم تشغيله
chmod +x setup_and_run.sh
./setup_and_run.sh
```
