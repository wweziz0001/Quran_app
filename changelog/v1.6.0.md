# v1.6.0 - Microservices Scaffold (Stage 2)

## 📋 Overview

This release implements **Stage 2** of the Enterprise Platform upgrade, transforming the application from a monolithic architecture to a microservices architecture.

## 🚀 New Features

### Microservices Architecture

Transformed the application into 8 independent microservices:

| Service | Port | Description |
|---------|------|-------------|
| `quran-service` | 3001 | Surahs, Ayahs, Mushafs management |
| `audio-service` | 3002 | Reciters, Recitations, Audio streaming |
| `search-service` | 3003 | Search, Elasticsearch integration |
| `tafsir-service` | 3004 | Tafsir, Translations |
| `users-service` | 3005 | Auth, Users, Sessions |
| `reciter-service` | 3006 | Reciters management |
| `ai-service` | 3007 | AI embeddings, LLM features |
| `admin-service` | 3008 | Admin panel, database management |

### Shared Module

- **`services/shared/`** - Shared utilities, types, and database connection
  - `db.ts` - Prisma client singleton
  - `types.ts` - Common TypeScript interfaces
  - `middleware.ts` - CORS, rate limiting, logging
  - `utils.ts` - Utility functions

### Kubernetes Support

- **`k8s/namespace.yaml`** - Kubernetes namespace
- **`k8s/configmaps/`** - Application configuration
- **`k8s/secrets/`** - Sensitive data management
- **`k8s/deployments/`** - Deployment manifests for all services
- **`k8s/services/`** - Service definitions
- **`k8s/ingress.yaml`** - Ingress routing

### Docker Support

- Each service has its own `Dockerfile`
- **`docker-compose.yml`** for local development

## 🔧 Technical Details

### Services Structure

Each microservice follows a consistent structure:
```
services/
├── [service-name]/
│   ├── Dockerfile
│   ├── package.json
│   └── src/
│       ├── index.ts
│       └── routes/
```

### Technologies Used

- **Hono** - Fast web framework for services
- **z-ai-web-dev-sdk** - AI integration (ai-service)
- **bcryptjs** - Password hashing (users-service)
- **jsonwebtoken** - JWT authentication (users-service)

## 🔄 Architecture Change

### Before (Monolith)
```
┌─────────────────────────────────────┐
│         Next.js Application          │
│  ┌─────────┐ ┌─────────┐ ┌────────┐ │
│  │  API    │ │  UI     │ │ Admin  │ │
│  │ Routes  │ │ Pages   │ │ Panel  │ │
│  └─────────┘ └─────────┘ └────────┘ │
│  ┌─────────────────────────────────┐│
│  │      Prisma + SQLite            ││
│  └─────────────────────────────────┘│
└─────────────────────────────────────┘
```

### After (Microservices)
```
                    ┌──────────────┐
                    │   Gateway    │
                    │   (Caddy)    │
                    └──────┬───────┘
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
   ┌────▼────┐       ┌─────▼─────┐      ┌────▼────┐
   │ quran-  │       │  audio-   │      │ search- │
   │ service │       │  service  │      │ service │
   └────┬────┘       └─────┬─────┘      └────┬────┘
        │                  │                  │
   ┌────▼────┐       ┌─────▼─────┐      ┌────▼────┐
   │ tafsir- │       │  users-   │      │  ai-    │
   │ service │       │  service  │      │ service │
   └────┬────┘       └─────┬─────┘      └────┬────┘
        │                  │                  │
        └──────────────────┼──────────────────┘
                           │
                    ┌──────▼──────┐
                    │   SQLite    │
                    │  (Shared)   │
                    └─────────────┘
```

## 📝 Notes

- Services use shared Prisma client from `services/shared/db.ts`
- Health check endpoints available on `/health` for all services
- CORS enabled on all services
- Logging middleware included

## ⬆️ Upgrade Notes

This is a major architectural change. The main Next.js application remains unchanged and continues to serve the UI. The microservices are optional and can be used independently or alongside the main application.

To use the microservices:
1. Build and run using `docker-compose up`
2. Or deploy to Kubernetes using the manifests in `k8s/`
3. Services will be available on their respective ports
