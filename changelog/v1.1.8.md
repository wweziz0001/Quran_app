# v1.1.8 - Bug Fixes

## Bug Fixes

### Fixed: Admin Panel Navigation Issue
- **Problem**: After performing any operation in the admin panel (add, edit, delete), the page would redirect to the Dashboard instead of staying on the current section.

- **Root Cause**: `AlertDialogAction` component from Radix UI has a default behavior that can cause unexpected page reloads or state resets.

- **Solution**: Replaced all `AlertDialogAction` components with regular `Button` components and added `e.preventDefault()` to delete handlers.

## Technical Details

The issue was caused by how `AlertDialogAction` from Radix UI handles click events. It automatically closes the dialog and may trigger form submission or other default behaviors that can cause the page to refresh or state to reset.

The fix involves:
1. Using regular `Button` components instead of `AlertDialogAction`
2. Adding `e.preventDefault()` to prevent default click behavior
3. Manually managing dialog state with `setDeleteDialogOpen(false)`

This ensures that:
- Delete operations complete successfully
- The user stays on the current admin section
- The page doesn't refresh or redirect unexpectedly
