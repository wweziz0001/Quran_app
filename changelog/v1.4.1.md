# [1.4.1] - 2025-01-16

## 🔧 الإصلاحات

### إصلاح تبويب Schema في Database Management

**المشكلة:**
تبويب Schema في Database Management لا يظهر محتويات الجداول ولا العلاقات. كانت الجداول تظهر كـ nodes فارغة بدون أي حقول.

**السبب:**
API `/api/admin/db/tables` كان يعيد `columns: []` فارغة دائماً. هذا حدث بسبب تبسيط سابق للـ API لحل مشكلة البطء، حيث تمت إزالة جلب الـ columns من قاعدة البيانات.

**الحل:**
تحديث API ليعيد الـ columns من **Prisma Schema** بدلاً من قاعدة البيانات مباشرة (أسرع بكثير):
- قراءة `prisma/schema.prisma` وتحليله برمجياً
- استخراج الحقول مع أنواعها
- تحديد Primary Keys من `@id`
- استخراج Foreign Keys من علاقات `@relation`
- تحديد Nullable من وجود `?` في الحقل

**الملفات المتأثرة:**
- `src/app/api/admin/db/tables/route.ts` - إعادة كتابة كاملة للدالة `parsePrismaSchema()`

**التفاصيل:**
```diff
// قبل: columns فارغة
{
  name: "Ayah",
  columns: [],  // فارغة!
  rowCount: 0
}

// بعد: columns كاملة مع foreign keys
{
  name: "Ayah",
  columns: [
    { name: "id", type: "Int", primaryKey: true, nullable: false },
    { name: "surahId", type: "Int", nullable: false,
      foreignKey: { table: "Surah", column: "id" } },
    { name: "ayahNumber", type: "Int", nullable: false, primaryKey: false },
    // ...
  ],
  rowCount: 204
}
```

**النتيجة:**
- ✅ تبويب Schema يعرض **31 جدول** مع جميع الحقول
- ✅ **23 علاقة** تظهر بشكل صحيح بين الجداول
- ✅ Primary Keys محددة بأيقونة المفتاح
- ✅ Foreign Keys محددة بأيقونة الرابط
- ✅ أداء سريع (48ms بدلاً من استعلامات متعددة لكل جدول)
